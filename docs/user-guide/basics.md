# User Guide Overview

This page serves as the entry point to the User Guide. We recommend reading the following pages in order to systematically understand Aspara's features.

## Recommended Reading Order

- [Core Concepts](concepts.md)
  Explains the core concepts of Aspara including Projects, Runs, and Metrics, along with the overall architecture.
- [Metadata and Notes](metadata.md)
  Explains how to manage metadata such as tags and notes from code and the dashboard.
- [Best Practices](best-practices.md)
  Covers useful patterns for production use, including naming conventions for runs and metrics, and logging frequency.
- [Troubleshooting](troubleshooting.md)
  Provides solutions for common issues like logs not being saved or not appearing in the dashboard.

## Advanced Topics (Read as Needed)

- [Configuration (Storage Location, etc.)](../advanced/configuration.md)
  Explains how to customize Aspara's behavior through environment variables and code, including data directories and storage methods.
- [Local vs Remote Tracking](../advanced/local-vs-remote.md)
  Explains the differences between local and remote tracking modes, and when to use each.
- [Storage](../advanced/storage.md)
  Covers storage backends such as JSONL and Polars, and their characteristics.
